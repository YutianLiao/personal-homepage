# Start Here: Why I Keep a Plain-Text Site

TL;DR: A markdown-first site removes friction between thinking and publishing.

I want one place where learning artifacts can be indexed and revisited.
Plain text is durable, grep-friendly, and easy to version.

## What this site optimizes for

- Fast writing and easy editing.
- Stable links across pages.
- Clear hierarchy over visual complexity.

## Writing conventions

- Every page should be skimmable in under one minute.
- Every link should include one-line context.
- Every long note should start with a short TL;DR.
